{% if main_role == 'Project Manager' %}

Software project managers provide the consistency needed to see software projects through from start to finish. With a mixture of technical knowledge and leadership, they oversee both the product being produced and the people performing the work. Software project managers generally work full time in an office environment, though performing extra hours is commonplace when problems need resolution or deadlines near. After years of experience, software project managers may move on to positions such as senior product manager or software engineering manager.

{% endif %} {% if main_role == 'Lead Programmer' %}

Lead programmers are responsible for managing and overseeing the work of other programmers. They commonly act as a liaison between developers and management, ensuring that everyone is on the same page with regard to project goals and objectives. Lead programmers may also be responsible for planning out projects in advance, breaking them down into smaller tasks that individual developers can tackle. This helps ensure that all parties involved have a clear idea of what needs to be done and when it needs to be done by.

{% endif %} {% if main_role == 'UI/UX Designer' %}

UI/UX designers create the user interface for an app, website, or other interactive media. Their work includes collaborating with product managers and engineers to gather requirements from users before designing ideas that can be communicated using storyboards. They also process flows or sitemaps. They'll then use their creativity while applying color palettes based on what best fits into a particular design conceptually.

{% endif %} {% if main_role == 'Quality Assurance Engineer' %}

The primary responsibility of the QA Engineer is to prevent defects. In addition, they are responsible for identifying and eliminating any mistakes in code before its release to customers. Such errors could result from various issues such as insufficient data input or an improperly designed interface element.

{% endif %} {% if main_role == 'Business Analyst' %}

A business analyst is a member of a product development team who analyzes the business domain, documents its processes and systems, outlines business requirements, and matches a software business model with the software being built.

{% endif %}

Purpose of this system:

This system will predict the main software engineering role as well as the secondary software engineering role of a student studying at Western Mindanao State University-College of Computing Studies. Furthermore, this system will also allow instructors to generate a group of students who are taking the software engineering course.

Process:

  • To start, click on the "Start Prediction" button above.
  • The system will then request the student to input their grades in respective major courses.It will then output the GPA of the student.
  • The system will now ask the students to take the MBTI personality type test.
  • The system will now ask the students to take the multiple intelligence test.
  • The system will fetch the necessary data which includes the GPA, the personality type of the student, and the multiple intelligence test result. After that, the system will process the inputted data and outputs the predicted main and secondary role.

Software Engineering Roles:

  • Project Manager
  • Lead Programmer
  • UI/UX Designer
  • Business Analyst
  • Quality Assurance Engineer
